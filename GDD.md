# Sword & Glory — Game Design Document

**Platform:** Web browser (single HTML file), network multiplayer via Firebase  
**Visual style:** Pixel art (prototype: canvas shapes)  
**Session length:** ~20 minutes  
**Players:** 2 (simultaneous turns)

---

## Design Philosophy

- Simple rules, emergent complexity
- Complexity lives in decisions before resolution, not in resolution itself
- Information asymmetry creates mastery curve
- Every fight is a resource economy decision, not just a power check
- The game should never leave the "opening phase" feeling — always discovery, always tension
- Inspired by the best 20 minutes of HoMM and Civ: scouting, discovery, legendary run fantasy

---

## Win Condition

Reduce the enemy kingdom to 0 health.  
Each kingdom starts at **10 health**. Each lost War Party defense = **-1 health**.  
Map combat (POI capture) never affects kingdom health.

---

## Map

- Top-down **hexagonal grid** (pointy-top axial coordinates)
- Shape: **Star** — 6 arms extending from a radius-2 center cluster
  - Each arm is 3 hexes long (hex distances 3, 4, 5 from center)
  - Player 1 starts at **east arm end** (5, 0)
  - Player 2 (AI) starts at **west arm end** (−5, 0)
  - NE, SW, NW, SE arms contain POIs only
- **Fog of war** — HoMM style: once revealed stays revealed forever; opponent spells can cover it back
- **Road** connects both cities through the east arm, center, and west arm — revealed at game start, drawn as a visible dirt path
- Both cities and the road are visible from turn 1
- **POIs (Points of Interest)** are guarded by neutral stacks; strength shown as estimated range `~N`
- POIs on the road: (3, 0) near player, (−3, 0) near AI
- Center (0, 0) is highest-value POI — most contested
- Army can move **3 hexes per turn**, BFS-based (uniform terrain cost)
- **Fire tiles** — set by Meteor; block army movement that turn; cleared next turn

---

## Turns

- **Simultaneous** — all players act during the same turn window
- Turn advances only after **all players end their turn**
- UI: "End Turn" → button becomes "Waiting for opponent..." (disabled)
- Prototype: AI opponent simulated with a 1.5s delay (`LOCAL_MODE = true`)
- Firebase multiplayer: waiting phase gates on opponent's signal

---

## Resources

### Gold
- Base income: **+5 per turn**
- Spent on: sending War Party (1g), buildings, Training Ground upgrades
- Generated by: POIs, Marketplace building, Transmute spell

### Mana
- Unlocked by: School building
- Base income: +2/turn from School
- Spent on: casting spells

### Glory Coins (✦)
- Earned by: **winning** an attack or defense against enemy kingdom (1 per win — either attacker or defender can earn)
- Spent on:
  - Kingdom-wide +1 ATK: cost = 1 + existing ATK upgrades (G-ATK)
  - Kingdom-wide +1 DEF: cost = 1 + existing DEF upgrades (G-DEF)
  - *(planned)* Improving a unit type's unique ability
- Glory upgrades apply retroactively to all cards in hand
- Glory Coin upgrade costs are **independent** from Training Ground upgrade costs

## Card Stat Model

Each unit card tracks stat components separately:

| Component | Field | Source |
|---|---|---|
| Base ATK/DEF | `baseAtk` / `baseDef` | Unit definition |
| T-ATK / T-DEF | `tAtk` / `tDef` | Training Ground upgrades |
| G-ATK / G-DEF | `gAtk` / `gDef` | Glory Coin upgrades |
| **F-ATK / F-DEF** | `atk` / `def` | Sum of all above — what combat uses |

`recompute(card)` recalculates F-ATK and F-DEF whenever any component changes.  
Spells can target any specific component layer.

## Army & Cards

- Units are cards with ATK, DEF, tier, cost, tags, and a skill slot
- Card order matters — units proc left to right within each phase
- **Townhall** draws 3 random cards from the player's unit pool each turn
- Cards accumulate in hand; committed to attack/defense per War Party

---

## Combat System

### Map Combat (POI Capture)
- Army total ATK must exceed stack DEF to capture
- All units that participated **die next turn**
- POI always drops **at least 1 unit** as reward
- If Meteor reduces a stack's DEF to ≤ 0, the stack dies without combat
- **Does NOT affect kingdom health**

### War Party (Kingdom Attack)
- Player spends **1 gold** to send a War Party to the enemy kingdom
- Defender sets up defense for **free** (no cost)

#### Combat Phases (shared `runPhases()` resolver)
1. **Buff Phase** — `passive` skill slots fire (both sides simultaneously, left-to-right)
2. **Pre-Battle Phase** — `skirmish` skill slots fire on the attacking side
   - Attacker base Speed: 6 | Defender base Speed: 5 → attacker goes first by default
   - Slow debuff: −2 Speed to target army (planned)
   - If attacker Speed ≤ defender Speed → defender pre-battles first (planned)
   - **Anti-projectile** check after all skirmish skills: halves Projectile debuf if defender has a Knight
3. **Combat** — `combat` skill slots fire; compare total F-ATK vs total F-DEF
   - Attacker wins → defender loses 1 kingdom health; attacker earns +1 Glory Coin
   - Attacker loses → defending army revealed and disappears next turn; defender earns +1 Glory Coin

#### Asymmetry
- Attacking costs 1 gold; defending is free but reveals and expires next turn
- Neither side is strictly better — intentional design

#### Flat Damage Rule
- Every lost fight = exactly **1 health**, regardless of army size
- Health tracks decision quality, not power

| Unit | Cost | ATK | DEF | Tags | Skill |

## Session Context (for future Claude)

### Architecture Summary
- **Single file**: `/home/user/SwordAndMagic/index.html` (~1700 lines)
- **Key state**: `turn`, `gold`, `health`, `enemyHealth`, `army {q,r,movesLeft}`, `hand[]`, `phase`, `unitPool[]`, `buildings Set`, `gloryCoins`, `gloryUpgrades {atk,def}`, `trainingLevels {}`, `fireTiles Set`, `playerSpells[]`
- **AI state**: `ai { gold, goldIncome, hand, unitPool, buildings, attackCD }`
- **`LOCAL_MODE = true`**: AI simulated; set to `false` for Firebase

### Combat Architecture
- `runPhases(atkCards, defCards, speedNote)` — shared phase resolver; always pass **deep copies** (`.map(c => ({...c}))`)
- Skill slots on `UNIT_DEFS[name]`: `passive({ self, allies, idx, lines, tag })`, `skirmish({ self, allies, cbt })`, `antiProjectile: true`, `combat({ self, cbt })`
- `cbt` object: `{ defCards, defDebuf, lines, tag }` — mutated during skirmish phase
- `buildCombatLog(attackers, defenders)` — player attacks AI; uses `runPhases`
- `buildDefenseCombatLog(aiForce, playerDef)` — AI attacks player; uses `runPhases`

### Card Model
```
baseAtk + tAtk + gAtk = atk (F-ATK)   — recompute(c) keeps in sync
baseDef + tDef + gDef = def (F-DEF)
```
